## 📌 第一步：打开 GitHub Actions 页面

### 方法 1：直接点击链接（推荐）

### 方法 2：手动输入网址

1. 打开浏览器（Chrome/Edge/Firefox 等）
2. 在地址栏输入：
   ```
   https://github.com/lover520f/ebox/actions
   ```

## 📌 第二步：登录 GitHub（如未登录）

如果你看到登录页面：

1. 输入你的 GitHub 用户名/邮箱
2. 输入密码
3. 完成两步验证（如已开启）
4. 点击 "Sign in"

---

## 📌 第三步：找到工作流列表

打开 Actions 页面后，你会看到：

### 左侧边栏
```
Workflows (2)
├── Build Windows Release
└── Windows Build and Release
```

### 中间区域
可能显示：
- "No runs yet"（还没有运行）
- 或之前的运行记录

---

## 📌 第四步：选择工作流

**点击左侧的**: `Build Windows Release`

（这是简化版工作流，推荐使用）

---

## 📌 第五步：点击 "Run workflow" 按钮

进入工作流页面后：

1. **找到页面右侧或顶部的绿色按钮**:
   ```
   [Run workflow] ▼
   ```

2. **点击这个按钮**，会弹出一个窗口

---

## 📌 第六步：配置工作流参数

在弹出的窗口中：

### Branch（分支）
- 选择：`master`（默认）

### Version（版本号）
- 输入：`v1.0.0`

### 确认信息
```
Branch: master ▼
Version: v1.0.0
```

### 点击运行
点击绿色的 **"Run workflow"** 按钮

---

## 📌 第七步：等待构建启动

点击后你会看到：

1. 页面自动刷新
2. 出现一个新的运行记录
3. 状态显示为：
   - 🟡 "Pending"（等待中）
   - 🟠 "Queued"（已排队）
   - 🟢 "In progress"（进行中）

---

## 📌 第八步：实时查看构建进度

## 🔍 故障排查

### 问题 1: 看不到 "Run workflow" 按钮

**原因**: 工作流可能未激活

**解决方法**:
1. 刷新页面（F5 或 Ctrl+R）
2. 确认你已登录 GitHub
3. 检查仓库是否是私有仓库

### 问题 2: 工作流失败

**查看错误**:
1. 点击失败的运行记录（红色 X）
2. 点击失败的步骤
3. 查看错误日志

**常见原因及解决**:

| 错误 | 解决方法 |
|------|----------|
| Flutter 安装失败 | 点击 "Re-run jobs" 重试 |
| 依赖安装失败 | 检查网络，重试 |
| 构建失败 | 查看具体错误信息，可能需要修复代码 |
| Release 上传失败 | 检查仓库权限 |

### 问题 3: 构建时间超过 20 分钟

**可能原因**:
- GitHub Server 拥挤
- 网络问题
- 构建配置需要优化

**解决方法**:
1. 等待 5 分钟看是否继续
2. 取消后重试
3. 查看日志是否有卡住的步骤
